=== Graph RAG

Implement retrieval-augmented generation (RAG) that combines vector similarity, knowledge graph traversal, and full-text search to retrieve richer, more connected context for LLM queries -- all within a single database.

==== ArcadeDB Features Used

* **<<graph-model,Graph Traversal>>** -- Multi-hop entity bridging across knowledge graph relationships
* **<<vector-model,Vector Similarity>>** -- Semantic chunk retrieval using `vectorNeighbors()` with LSM_VECTOR indexes
* **<<searchengine-model,Full-Text Search>>** -- Keyword-based chunk lookup
* **Neo4j Bolt Protocol** -- Driver compatibility via port 7687 for LangChain4j integration

==== Architecture Overview

[cols="1,3"]
|===
| Vertices | `Chunk` (content, source, chunkIndex, embedding), `Entity`, `Person`, `Concept`, `Organization`
| Edges | `MENTIONS`, `RELATES_TO`, `WORKS_AT`, `AUTHORED`
|===

Document chunks carry embedding vectors and link to extracted entities through `MENTIONS` edges. Entities connect via `RELATES_TO`, enabling multi-hop discovery that bridges chunks from different documents through shared entity mentions.

==== Key Queries

*Hybrid Vector + Graph Search* -- Find semantically similar chunks and expand through entity connections:

[source,sql]
----
SELECT content, source FROM (
  SELECT vectorNeighbors('Chunk[embedding]', [0.9, 0.1, 0.8, 0.2], 5) AS neighbors
)
----

*Multi-Hop Entity Bridge* -- Discover related entities across documents:

[source,cypher]
----
MATCH (c:Chunk)-[:MENTIONS]->(e:Entity)-[:RELATES_TO*1..2]-(related:Entity)<-[:MENTIONS]-(other:Chunk)
WHERE c.source = 'quantum_computing.txt'
RETURN related.name, other.content, other.source
----

*Composite Scoring* -- Combine vector distance with graph connectivity for ranked retrieval:

[source,sql]
----
SELECT content, source,
  (1.0 / (1.0 + distance)) * 0.7 + (entityCount / 5.0) * 0.3 AS compositeScore
FROM ChunkScores ORDER BY compositeScore DESC
----
