==== Database Panel

The Database Panel shows the information about the selected database and its schema.

image::studio-database.png[]

The Database Panel is organized into three tabs:

* *Schema* - View and manage vertex, edge, and document types
* *Settings* - View and modify database configuration settings
* *Backup* - View backup configuration and available backups

[discrete]
===== Database Commands

The toolbar at the top provides quick actions:

* *Create* to create a new database.
Enter the database name in the popup and the new database will be ready to be used
* *Drop* to drop the current database.
*_NOTE: This operation cannot be undone._*
* *Reset* to drop all data and schema from the database.
* *Backup* to execute an immediate backup of the selected database.
The backup will be available under the directory `backups` where ArcadeDB server is installed.
For more information look at <<backup,Backup>>.

[discrete]
===== Schema

The Schema tab displays all types defined in the database, organized by category:

* *Vertices* - Shown with colored badges (e.g., blue, purple, orange) with the type name and record count
* *Edges* - Shown with colored badges with the type name and record count

Click on any type badge to view its detailed schema information in the right panel, including:

* Configured properties with their data types
* Defined indexes
* Quick actions to display records of that type

image::studio-database-schema.png[]

[discrete]
===== Database Settings

image::studio-database-settings.png[]

The Settings tab displays the current database configuration parameters.
You can view and modify settings that control database behavior.

[[studio-database-backup]]
===== Database Backup Tab

The Backup tab in the Database Panel shows backup information specific to the selected database.

image::studio-database-backup.png[Database Backup Tab]

The Backup tab displays:

* *Backup Configuration* - Shows the current backup schedule and retention settings for this database
* *Statistics* - Total number of backups and disk space used
* *Available Backups* - List of all backup files with timestamps and file sizes

From this tab you can:

* Click *Refresh* to reload the backup list
* Click *Backup Now* to trigger an immediate backup of the current database
* View backup history and verify successful backups
