== Connect from your application

ArcadeDB exposes several wire protocols and drivers, so you can keep your existing client libraries:

* xref:reference/http-api/http.adoc[HTTP / JSON REST API] &mdash; language-agnostic, no driver required
* xref:how-to/connectivity/postgres.adoc[PostgreSQL wire protocol] &mdash; existing Postgres clients connect unchanged
* xref:how-to/connectivity/bolt.adoc[Neo4j Bolt protocol] &mdash; reuse Neo4j's official drivers
* xref:reference/mongodb-ql/mongo.adoc[MongoDB QL] &mdash; MongoDB drivers
* xref:reference/redis-ql/redis.adoc[Redis commands] &mdash; Redis clients
* xref:reference/mcp/mcp.adoc[MCP server] &mdash; for AI agents and LLM tooling
* xref:reference/java-api/java-reference.adoc[Native Java API] &mdash; embedded mode
* xref:how-to/connectivity/jdbc.adoc[JDBC driver] &mdash; for BI and traditional SQL tools

== Pick a query language

ArcadeDB supports six query languages over the same data &mdash; use whichever your team already knows:

* xref:reference/sql/sql-introduction.adoc[SQL] &mdash; extended with graph, full-text, and vector functions
* xref:reference/cypher/cypher-introduction.adoc[Cypher (openCypher)] &mdash; drop-in replacement for Neo4j Cypher
* xref:reference/gremlin/gremlin.adoc[Apache TinkerPop Gremlin]
* xref:reference/graphql/graphql.adoc[GraphQL]
* xref:reference/mongodb-ql/mongo.adoc[MongoDB QL]
* xref:reference/redis-ql/redis.adoc[Redis commands]
